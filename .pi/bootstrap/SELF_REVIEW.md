# 自我复盘机制

## 频率
- **日度**：检查昨日决策是否正确（盘后15:30执行）
- **周度**：汇总本周所有决策，统计准确率
- **月度**：深度复盘，识别系统性偏差

## 复盘模板

### 决策准确性评估
| 决策ID | 股票 | 决策 | 当时价 | 7日后价 | 方向正确？ | 教训 |
|--------|------|------|--------|---------|-----------|------|
| #N | XXX | 买入/回避 | ¥XX | ¥XX | ✅/❌ | ... |

### 系统性偏差检查
- [ ] 是否过度保守（回避太多，买入太少）？
- [ ] 是否在顶部追涨、在底部恐慌？
- [ ] 是否忽略了周期位置？
- [ ] 是否在工具失败后投降而不是手动计算？
- [ ] 是否主动巡检了持仓？
- [ ] 是否记录了每个决策？

## 错误分类

| 错误类型 | 描述 | 纠正方法 |
|---------|------|---------|
| **A类：过度保守** | 看到风险就回避，错过机会 | 强制填写决策框架第1条 |
| **B类：追涨杀跌** | 涨了才推荐买入，跌了才推荐卖出 | 检查决策时的技术位置 |
| **C类：忽略周期** | 把周期低盈利当作永久性衰退 | 强制框架第2条 |
| **D类：工具投降** | 工具失败后放弃分析 | 降级手动计算 |
| **E类：被动等待** | 不主动巡检组合 | 每日cron强制执行 |

## 复盘存储

- 日度复盘：记录到 `.pi/reviews/YYYY-MM-DD.md`
- 周度/月度：记录到 `.pi/reviews/weekly/` 和 `.pi/reviews/monthly/`
- 关键教训：提炼后写入 memory 系统

## 触发机制

- 用户主动要求："复盘一下最近的决策"
- 定时触发：每日15:30后首次对话
- 重大失误后：立即复盘，不等定时
