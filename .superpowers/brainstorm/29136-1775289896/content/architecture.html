架构设计

第一节：数据流 + 模块结构

数据流

Pi 运行时
FunctionHook: SessionStart → logger.write(session_start)
FunctionHook: UserPromptSubmit → logger.write(user_prompt)
FunctionHook: PreToolUse → logger.write(tool_call + ts)
FunctionHook: PostToolUse → logger.write(tool_result + duration_ms)
FunctionHook: PostToolUseFailure → logger.write(tool_error)
FunctionHook: Stop → logger.write(session_end + messages[])
写入
.pi/logs/session-<id>.jsonl  ← 每行一个事件，追加写
Stop 时生成报告
htmlReport.generate(jsonl) → .pi/logs/session-<id>.html

新增文件

src/observability/
logger.ts — 写 JSONL，管理当前 session 文件路径
htmlReport.ts — 读 JSONL → 生成自包含 HTML
tokenCount.ts — 复用 roughTokenCount，统计每轮 token 数
index.ts — 导出 initObservability()，注册所有 hooks

JSONL 事件格式（每行一个 JSON）

{"event":"session_start","session_id":"abc123","cwd":"/proj","ts":1000}
{"event":"user_prompt","prompt":"帮我写排序函数","ts":1001}
{"event":"tool_call","tool":"Read","input":{"file_path":"src/utils.ts"},"ts":1005}
{"event":"tool_result","tool":"Read","result":"export...","duration_ms":45,"ts":1006}
{"event":"tool_call","tool":"Write","input":{...},"ts":1008}
{"event":"tool_result","tool":"Write","result":"ok","duration_ms":12,"ts":1009}
{"event":"session_end","stop_reason":"done","messages":[...],"ts":1020}

集成方式

在 src/screens/REPL.tsx 的初始化 useEffect 里，和 initSessionMemory() 并列调用 initObservability()。一次注册，永久生效。
