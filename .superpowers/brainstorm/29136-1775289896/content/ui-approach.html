可视化 UI 的形态

Pi agent 日志查看器应该以什么形式存在？

A

独立 Web 应用

本地启动一个 HTTP 服务（Node.js），浏览器打开 localhost:port 查看。日志保存为 JSONL 文件，Web UI 读取展示。

优点

  • 富交互（React/Vue）
  • 与 Pi 完全解耦
  • 可扩展为多 session 对比

缺点

  • 需要单独启动服务
B

静态 HTML 报告

每次 session 结束后，自动生成一个独立的 session-xxx.html 文件，双击浏览器直接打开，无需服务器。

优点

  • 零依赖，随时可看
  • 可分享给他人
  • 可归档

缺点

  • 多 session 对比不方便
  • 不能实时追加
C

Web 应用 + 自动启动

Pi 启动时自动在后台启动日志服务器，session 结束后浏览器自动打开查看页面。类似 vitest UI 或 webpack-bundle-analyzer 的体验。

优点

  • 零配置体验
  • 最流畅的工作流
  • 支持历史 session 浏览

缺点

  • 实现最复杂
